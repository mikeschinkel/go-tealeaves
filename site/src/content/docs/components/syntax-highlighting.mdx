> **Module:** `teahilite` · **Type:** `Highlighter` · **Install:** `go get github.com/mikeschinkel/go-tealeaves/teahilite`

`teahilite` renders syntax-highlighted code as ANSI escape sequences suitable for embedding in Bubble Tea views. It wraps the [Chroma](https://github.com/alecthomas/chroma) library with a simple API — choose your style and formatter once, then call `Highlight()` on each piece of code.

<Aside type="caution">
This module is under active development. API may change.
</Aside>

## Install

```bash
go get github.com/mikeschinkel/go-tealeaves/teahilite
```

## Quick example

```go
import "github.com/mikeschinkel/go-tealeaves/teahilite"

// Package-level convenience (uses monokai + terminal256 defaults)
highlighted, err := teahilite.Highlight("package main\n\nfunc main() {}", "go")

// Or create a configured Highlighter instance
h := teahilite.NewHighlighter(teahilite.HighlighterArgs{
    StyleName: "dracula",
})
highlighted, err = h.Highlight(code, "go")
lines, err := h.HighlightLines(code, "go") // []string, one per line
```

## Highlighter type

`Highlighter` holds pre-built Chroma style and formatter objects. Create once and reuse across frames to avoid repeated style lookups:

```go
h := teahilite.NewHighlighter(teahilite.HighlighterArgs{
    StyleName:     "monokai",    // default
    FormatterName: "terminal256", // default
})

// Returns ANSI-styled string, error
result, err := h.Highlight(code, language)

// Returns []string, one element per line
lines, err := h.HighlightLines(code, language)
```

### HighlighterArgs

| Field | Default | Description |
|-------|---------|-------------|
| `StyleName` | `"monokai"` | Chroma style name. Falls back to Chroma's fallback style if unknown. |
| `FormatterName` | `"terminal256"` | Chroma formatter name. Falls back to Chroma's fallback formatter if unknown. |

### Highlighter API

| Method | Returns | Description |
|--------|---------|-------------|
| `Highlight(code, language string)` | `(string, error)` | Full highlighted output as a single ANSI string |
| `HighlightLines(code, language string)` | `([]string, error)` | Same, split on newlines — one element per line |

## Package-level functions

Package-level functions use a shared default `Highlighter` (monokai + terminal256). These are convenient for one-off use but cannot be configured:

```go
result, err := teahilite.Highlight(code, "python")
lines, err := teahilite.HighlightLines(code, "python")
```

## Theme selection

Themes are Chroma style names. Common options:

| Style | Description |
|-------|-------------|
| `"monokai"` | Dark, high-contrast (default) |
| `"dracula"` | Dark with purple accents |
| `"github"` | Light, GitHub-style |
| `"solarized-dark"` | Solarized dark |
| `"solarized-light"` | Solarized light |
| `"vim"` | Classic Vim colors |

Unknown style names fall back to Chroma's built-in fallback style without error.

## DetectLanguage

`DetectLanguage` infers the Chroma lexer alias from a file path or name by matching its extension. The generic constraint accepts `dt.Filepath`, `dt.RelFilepath`, `dt.Filename`, or plain `string`. Returns `"text"` when no lexer matches.

```go
func DetectLanguage[S ~string](path S) (name string)
```

```go
lang := teahilite.DetectLanguage("main.go")        // "go"
lang  = teahilite.DetectLanguage("schema.sql")     // "sql"
lang  = teahilite.DetectLanguage("app.ts")         // "typescript"
lang  = teahilite.DetectLanguage("unknown.xyz")    // "text"

// Works with domain types from go-dt
lang  = teahilite.DetectLanguage(dt.Filepath("/src/main.py"))  // "python"
```

Combine with `Highlight` to avoid hard-coding language strings:

```go
h := teahilite.NewHighlighter(teahilite.HighlighterArgs{})
lang := teahilite.DetectLanguage(filePath)
result, err := h.Highlight(code, lang)
```

## Language detection

Pass the language name as a lowercase string. If the language is unknown or empty, Chroma's fallback lexer is used (plain text with no highlighting):

```go
h.Highlight(goCode, "go")
h.Highlight(tsCode, "typescript")
h.Highlight(sqlCode, "sql")
h.Highlight(unknownCode, "") // falls back to plain text
```

## Integration with SplitDiffModel

`teahilite` can supply the `HighlightFunc` for [`teadiff.SplitDiffModel`](/go-tealeaves/components/diff-viewer/):

```go
h := teahilite.NewHighlighter(teahilite.HighlighterArgs{StyleName: "dracula"})

diff := teadiff.NewSplitDiffModel(teadiff.SplitDiffModelArgs{
    HighlightFunc: func(text, lang string) string {
        result, _ := h.Highlight(text, lang)
        return result
    },
})
```

## API Reference

### Package-Level Functions

| Function | Signature | Description |
|----------|-----------|-------------|
| `Highlight` | `Highlight(code, language string) (string, error)` | Highlight code using the default `Highlighter` (monokai + terminal256); returns ANSI string |
| `HighlightLines` | `HighlightLines(code, language string) ([]string, error)` | Same as `Highlight` but split into one element per line |
| `DetectLanguage[S]` | `DetectLanguage[S ~string](path S) string` | Infer the Chroma lexer alias from a file path or name; returns `"text"` if no match |

### Highlighter Type

| Function | Signature | Description |
|----------|-----------|-------------|
| `NewHighlighter` | `NewHighlighter(args HighlighterArgs) *Highlighter` | Create a configured Highlighter instance |

### Highlighter Methods

| Method | Signature | Description |
|--------|-----------|-------------|
| `Highlight` | `Highlight(code, language string) (string, error)` | Return ANSI-styled string |
| `HighlightLines` | `HighlightLines(code, language string) ([]string, error)` | Return `[]string`, one element per line |

### HighlighterArgs Struct

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `StyleName` | `string` | `"monokai"` | Chroma style name |
| `FormatterName` | `string` | `"terminal256"` | Chroma formatter name |

### Theme Name Constants

| Const | Value | Description |
|-------|-------|-------------|
| `DefaultStyleName` | `"monokai"` | Default Chroma style |
| `DefaultFormatterName` | `"terminal256"` | Default Chroma formatter |

Common style names: `"monokai"`, `"dracula"`, `"github"`, `"solarized-dark"`, `"solarized-light"`, `"vim"`. Unknown names fall back to Chroma's built-in fallback style without error.
