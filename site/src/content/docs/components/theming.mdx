> **Module:** `teautils` · **Type:** `Theme` / `Palette` · **Install:** `go get github.com/mikeschinkel/go-tealeaves/teautils`

Tea Leaves provides a three-layer theming system that lets you apply consistent colors across all components:

1. **teacolor** — Named color constants (ANSI 256 + hex values)
2. **SystemPalette** — Semantic color slots (TextPrimary, Accent, StatusSuccess, etc.)
3. **Theme** — Component-specific style groups derived from a Palette

## Install

```bash
go get github.com/mikeschinkel/go-tealeaves/teautils
```

## Quick Start

```go
import "github.com/mikeschinkel/go-tealeaves/teautils"

// Use adaptive palette (auto-detects dark/light terminal)
theme := teautils.DefaultTheme()

// Apply to any component
grid := teagrid.NewGridModel(columns).WithTheme(theme)
statusBar := teastatus.NewStatusBarModel().WithTheme(theme)
modal := teamodal.NewModal(args).WithTheme(theme)
```

## Color Constants

The `teacolor` module provides named constants for all 256 ANSI colors plus curated semantic aliases:

```go
import "github.com/mikeschinkel/go-tealeaves/teacolor"

// ANSI 256 indexed colors
teacolor.Color0   // Black
teacolor.Color46  // Green

// Standard ANSI names (colors 0-15)
teacolor.Red, teacolor.BrightCyan

// Curated semantic names
teacolor.Coral, teacolor.SlateGray, teacolor.Teal
```

<Aside type="note" title="Package move">
`teacolor` was previously shipped as a sub-package of `teautils` (`teautils/teacolor`). It is now a standalone module at `github.com/mikeschinkel/go-tealeaves/teacolor`. Similarly, `teautils.SemanticColor` and `teautils.NewSemanticColor` are now type aliases that re-export from `teacolor` for backward compatibility — new code should import `teacolor` directly. See [Color Constants](/go-tealeaves/components/color-constants/) for the full `teacolor` API.
</Aside>

## Palette

A `SystemPalette` contains ~30 semantic color slots organized by purpose. Pass `nil` for fixed (non-adaptive) colors, or pass `&teautils.PaletteOpts{Adaptive: true}` to use ANSI 0-15 colors that respect the user's terminal theme:

```go
palette := teautils.DarkSystemPalette(nil)     // Dark terminal colors
palette := teautils.LightSystemPalette(nil)    // Light terminal colors
palette := teautils.AdaptiveSystemPalette(nil) // Auto-detect dark/light
palette := teautils.DefaultSystemPalette(nil)  // Alias for adaptive
```

Color slot groups:

| Group | Fields |
|-------|--------|
| Text hierarchy | `TextPrimary`, `TextSecondary`, `TextMuted`, `TextDim` |
| Accent colors | `Accent`, `AccentAlt`, `AccentSubtle` |
| Selection / focus | `SelectionBg`, `SelectionFg`, `FocusBorder`, `FocusBg` |
| Borders and chrome | `Border`, `BorderAccent`, `Separator` |
| Status indicators | `StatusSuccess`, `StatusInfo`, `StatusWarn`, `StatusError` |
| Interactive elements | `ButtonBg`, `ButtonFg`, `ButtonFocusBg`, `ButtonFocusFg` |
| Edit mode | `EditBg`, `EditFg` |
| Diff context | `TintPositive`, `TintNegative`, `TintNeutral` |
| Scrollbar | `ScrollTrack`, `ScrollThumb` |
| Syntax highlighting | `HighlightStyle` (Chroma style name, e.g. `"monokai"`, `"github"`) |

### Palette[T] — App Domain Colors

When your application needs colors beyond the system palette, embed `SystemPalette` inside the generic `Palette[T]`:

```go
type AppColors struct {
    Brand    teautils.SemanticColor
    Sidebar  teautils.SemanticColor
}

palette := teautils.Palette[AppColors]{
    System: teautils.DarkSystemPalette(nil),
    App: AppColors{
        Brand:   teautils.NewSemanticColor(teacolor.Color46),
        Sidebar: teautils.NewSemanticColor(teacolor.Color238),
    },
}
```

Pass `palette.System` to `teautils.NewTheme()` for component theming, and access `palette.App` directly for your own rendering code.

## Theme

A `Theme` derives lipgloss styles from a `SystemPalette` and organizes them by component. It also exposes a set of shared styles useful in custom component rendering:

```go
theme := teautils.NewTheme(palette)
// theme.Breadcrumb — BreadcrumbTheme
// theme.StatusBar  — StatusBarTheme
// theme.Modal      — ModalTheme
// theme.Grid       — GridTheme
// theme.Dropdown   — DropdownTheme
// theme.HelpVisor  — HelpVisorTheme
// theme.List       — ListTheme
```

Common shared styles available directly on `Theme`:

| Field | Type | Description |
|-------|------|-------------|
| `System` | `SystemPalette` | The source palette used to build the theme |
| `Border` | `lipgloss.Style` | Rounded border in the default border color |
| `BorderAccent` | `lipgloss.Style` | Rounded border in the accent border color |
| `Title` | `lipgloss.Style` | Bold accent-colored text for titles |
| `Message` | `lipgloss.Style` | Secondary-colored text for body messages |
| `Button` | `lipgloss.Style` | Unfocused button foreground style |
| `FocusedButton` | `lipgloss.Style` | Focused button with highlight foreground + background |
| `Item` | `lipgloss.Style` | Primary-colored list/grid item text |
| `SelectedItem` | `lipgloss.Style` | Selected item with selection foreground + background |
| `ActiveItem` | `lipgloss.Style` | Accent-colored text for the active/current item |

## WithTheme() on Components

Every Tea Leaves component supports `WithTheme()`:

| Component | Method |
|-----------|--------|
| teagrid | `model.WithTheme(theme)` |
| teastatus | `model.WithTheme(theme)` |
| teanotify | `model.WithTheme(theme)` |
| teamodal | `model.WithTheme(theme)` |
| teafields | `model.WithTheme(theme)` |
| teatree | `model.WithTheme(theme)` |
| teatext | `model.WithTheme(theme)` |
| teadiff | `model.WithTheme(theme)` |

<Aside type="tip">
`WithTheme()` is opt-in. Unthemed components use their existing default styles. Individual `With*Style()` methods still take precedence over theme styles.
</Aside>

## API Reference

### PaletteOpts

| Field | Type | Description |
|-------|------|-------------|
| `Adaptive` | `bool` | When `true`, selects ANSI 0-15 colors for text, accents, and status indicators — these respect the user's terminal theme. When `false` (default), all colors use fixed ANSI 16-255 values. |

### Palette Constructor Functions

| Function | Signature | Description |
|----------|-----------|-------------|
| `DarkSystemPalette` | `DarkSystemPalette(opts *PaletteOpts) SystemPalette` | Returns a palette suited for dark terminal backgrounds |
| `LightSystemPalette` | `LightSystemPalette(opts *PaletteOpts) SystemPalette` | Returns a palette suited for light terminal backgrounds |
| `AdaptiveSystemPalette` | `AdaptiveSystemPalette(opts *PaletteOpts) SystemPalette` | Detects terminal background; falls back to dark if detection fails |
| `DefaultSystemPalette` | `DefaultSystemPalette(opts *PaletteOpts) SystemPalette` | Alias for `AdaptiveSystemPalette` |

### Theme Constructor Functions

| Function | Signature | Description |
|----------|-----------|-------------|
| `NewTheme` | `NewTheme(sys SystemPalette) Theme` | Build a `Theme` from a `SystemPalette`, deriving all lipgloss styles |
| `DefaultTheme` | `DefaultTheme() Theme` | Returns `NewTheme(DefaultSystemPalette(nil))` |

### Component Theme Types

Each component-specific field on `Theme` is its own struct with exported `lipgloss.Style` fields:

#### BreadcrumbTheme (`theme.Breadcrumb`)

| Field | Description |
|-------|-------------|
| `ParentStyle` | Style for ancestor breadcrumb segments |
| `CurrentStyle` | Style for the current (last) segment |
| `SeparatorStyle` | Style for the separator between segments |
| `HoverStyle` | Style for a hovered/focused segment |

#### StatusBarTheme (`theme.StatusBar`)

| Field | Description |
|-------|-------------|
| `MenuKeyStyle` | Style for key labels in the status bar |
| `MenuLabelStyle` | Style for action labels in the status bar |
| `IndicatorStyle` | Style for status indicator text |
| `IndicatorSepStyle` | Style for separators between indicators |
| `BarStyle` | Base style for the full bar container |

#### HelpVisorTheme (`theme.HelpVisor`)

| Field | Description |
|-------|-------------|
| `TitleStyle` | Style for the visor title |
| `CategoryStyle` | Style for category headers |
| `KeyStyle` | Style for key labels |
| `DescStyle` | Style for key descriptions |

#### ModalTheme (`theme.Modal`)

| Field | Description |
|-------|-------------|
| `BorderStyle` | Rounded border style for the modal frame |
| `TitleStyle` | Style for the modal title |
| `MessageStyle` | Style for the modal body message |
| `ButtonStyle` | Style for unfocused buttons |
| `FocusedButtonStyle` | Style for the focused button |
| `CancelKeyStyle` | Style for the cancel key label |
| `CancelTextStyle` | Style for the cancel action text |

#### DropdownTheme (`theme.Dropdown`)

| Field | Description |
|-------|-------------|
| `BorderStyle` | Rounded border style for the dropdown |
| `ItemStyle` | Style for unfocused list items |
| `SelectedStyle` | Style for the selected/highlighted item |

#### ListTheme (`theme.List`)

| Field | Description |
|-------|-------------|
| `ItemStyle` | Style for regular list items |
| `SelectedItemStyle` | Style for the selected item |
| `ActiveItemStyle` | Style for the active (cursor) item |
| `FooterStyle` | Style for the list footer text |
| `StatusStyle` | Style for status messages |
| `EditItemStyle` | Style for an item in inline-edit mode |
| `ScrollbarStyle` | Style for the scrollbar track |
| `ScrollThumbStyle` | Style for the scrollbar thumb |

#### GridTheme (`theme.Grid`)

| Field | Description |
|-------|-------------|
| `HeaderStyle` | Style for column header cells |
| `BaseStyle` | Style for regular data cells |
| `HighlightStyle` | Style for the highlighted/selected row |
| `BorderStyle` | Style for the grid border |

### Exported Semantic Color Variables

`teautils` re-exports `SemanticColor` type aliases and pre-built variables from `teacolor` for use when constructing custom palettes. ANSI 0-15 named variables (terminal-theme-adaptive):

| Variable | ANSI Index |
|----------|-----------|
| `SemanticBlack` | 0 |
| `SemanticRed` | 1 |
| `SemanticGreen` | 2 |
| `SemanticYellow` | 3 |
| `SemanticBlue` | 4 |
| `SemanticMagenta` | 5 |
| `SemanticCyan` | 6 |
| `SemanticWhite` | 7 |
| `SemanticBrightBlack` | 8 |
| `SemanticBrightRed` | 9 |
| `SemanticBrightGreen` | 10 |
| `SemanticBrightYellow` | 11 |
| `SemanticBrightBlue` | 12 |
| `SemanticBrightMagenta` | 13 |
| `SemanticBrightCyan` | 14 |
| `SemanticBrightWhite` | 15 |

Additional pre-built fixed-color variables for commonly used ANSI 256 values (e.g., `SemanticColor46`, `SemanticColor178`, `SemanticColor240`) are available in the source; see `teautils/palette.go` for the full list. `SemanticColorNil` represents "no color" (transparent/default).
