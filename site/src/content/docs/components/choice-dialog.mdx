> **Module:** `github.com/mikeschinkel/go-tealeaves/teamodal` · **Type:** `ChoiceModel`

## Install

```bash
go get github.com/mikeschinkel/go-tealeaves/teamodal
```

Multi-option selection dialog with hotkey support and configurable horizontal or vertical layout. Each option can have a label, hotkey character, and string ID. Hotkeys trigger selection immediately without needing Tab + Enter.

## Quick Example

{/* verified: site/examples/choice-dialog/ — go test passes */}
```go
modal := teamodal.NewChoiceModel(&teamodal.ChoiceModelArgs{
    Title:   "How should we proceed?",
    Message: "There are unsaved changes in 3 files.",
    Options: []teamodal.ChoiceOption{
        {Label: "Save & Exit", Hotkey: 's', ID: "save"},
        {Label: "Discard",     Hotkey: 'd', ID: "discard"},
        {Label: "Continue",    Hotkey: 'c', ID: "continue"},
    },
    DefaultIndex: 0,
    Orientation:  teamodal.Horizontal,
})

// Open it
m.modal, cmd = m.modal.Open()

// In View()
if m.modal.IsOpen() {
    return m.modal.OverlayModal(backgroundView)
}

// Handle responses
case teamodal.ChoiceSelectedMsg:
    selectedID := msg.OptionID
    selectedIndex := msg.Index
case teamodal.ChoiceCancelledMsg:
    // User pressed Esc
```

## Features

- Horizontal or vertical option layout (`teamodal.Horizontal` / `teamodal.Vertical`)
- Hotkey-driven selection (case-insensitive) — single keypress selects immediately
- Tab/Arrow key navigation between options
- ANSI-aware overlay compositing via `OverlayModal()`
- Custom styling for title, message, buttons, border, and focus states
- Non-nil `tea.Cmd` message consumption signal (see [Modal Message Consumption](/go-tealeaves/guides/modal-consumption/))
- Auto-centering on screen based on `SetSize()` dimensions

## API Reference

### Constructor

| Function | Signature | Description |
|----------|-----------|-------------|
| `NewChoiceModel` | `NewChoiceModel(args *ChoiceModelArgs) ChoiceModel` | Create a multi-option choice dialog |

### Key Types

| Type | Description |
|------|-------------|
| `ChoiceModel` | Multi-option choice dialog with hotkeys |
| `ChoiceOption` | Option with `Label`, `Hotkey` (rune), and `ID` (string) fields |
| `Orientation` | `Horizontal` (default) or `Vertical` — controls button layout direction |
| `ChoiceModelArgs` | Full configuration struct — see fields below |

### ChoiceModelArgs fields

| Field | Type | Description |
|-------|------|-------------|
| `Title` | `string` | Optional title above message |
| `Message` | `string` | Main message text |
| `Options` | `[]ChoiceOption` | 2–5 options to display |
| `DefaultIndex` | `int` | Initially focused button (0-based) |
| `Orientation` | `Orientation` | `Horizontal` (default) or `Vertical` |
| `ShowBrackets` | `*bool` | Show `[ ]` around labels; `nil` = true for Horizontal, false for Vertical |
| `AllowCancel` | `*bool` | Allow Esc to cancel; `nil` defaults to `true` |
| `ShowCancelHint` | `*bool` | Show `[esc] Cancel` hint below options; `nil` follows `AllowCancel` |
| `ScreenWidth` | `int` | Screen width for centering |
| `ScreenHeight` | `int` | Screen height for centering |
| `BorderStyle` | `lipgloss.Style` | Optional border style override |
| `TitleStyle` | `lipgloss.Style` | Optional title style override |
| `MessageStyle` | `lipgloss.Style` | Optional message style override |
| `ButtonStyle` | `lipgloss.Style` | Optional unfocused button style override |
| `FocusedButtonStyle` | `lipgloss.Style` | Optional focused button style override |
| `CancelKeyStyle` | `lipgloss.Style` | Style for `[esc]` in the cancel hint |
| `CancelTextStyle` | `lipgloss.Style` | Style for `Cancel` text in the cancel hint |

### ChoiceKeyMap Fields

| Field | Default keys | Description |
|-------|-------------|-------------|
| `Confirm` | `enter` | Select the focused option |
| `Cancel` | `esc` | Cancel the dialog |
| `NextButton` | `tab`, `right`, `down` | Move to the next option |
| `PrevButton` | `shift+tab`, `left`, `up` | Move to the previous option |

| Function | Signature | Description |
|----------|-----------|-------------|
| `DefaultChoiceKeyMap` | `DefaultChoiceKeyMap() ChoiceKeyMap` | Return the default key bindings for `ChoiceModel` |

### Theming

| Method | Signature | Description |
|--------|-----------|-------------|
| `WithTheme` | `WithTheme(theme teautils.Theme) ChoiceModel` | Apply all modal styles from a `teautils.Theme` at once; sets `BorderStyle`, `TitleStyle`, `MessageStyle`, `ButtonStyle`, `FocusedButtonStyle`, `CancelKeyStyle`, and `CancelTextStyle` |

### Methods

| Method | Signature | Description |
|--------|-----------|-------------|
| `Open` | `Open() (ChoiceModel, tea.Cmd)` | Open the modal; pre-calculates position for overlay |
| `Close` | `Close() (ChoiceModel, tea.Cmd)` | Close the modal |
| `SetSize` | `SetSize(width, height int) ChoiceModel` | Set screen dimensions for centering |
| `IsOpen` | `IsOpen() bool` | Whether the modal is currently open |
| `FocusButton` | `FocusButton() int` | Index of the currently focused option |
| `OverlayModal` | `OverlayModal(background string) string` | Render modal centered over a background view |

### Messages

| Message | When sent |
|---------|-----------|
| `ChoiceSelectedMsg` | Choice option selected — contains `Index` (int) and `OptionID` (string) |
| `ChoiceCancelledMsg` | Choice modal cancelled with Esc |

## Configuration

{/* verified: site/examples/choice-dialog/ — go test passes */}
```go
modal := teamodal.NewChoiceModel(&teamodal.ChoiceModelArgs{
    Title:       "How should we proceed?",
    Message:     "Select an action.",
    Options:     options,
    Orientation: teamodal.Vertical,
    AllowCancel: boolPtr(true),
})

// Apply a theme (sets all style fields at once)
modal = modal.WithTheme(myTheme)
```

`ChoiceModel` uses the [Wither Methods](/go-tealeaves/guides/wither-methods/) pattern for configuration.
`WithTheme()` accepts a `teautils.Theme` and applies `theme.Modal.*` style fields. Individual `With*Style()` calls made after `WithTheme()` take precedence.

## Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `Tab` / `Right` / `Down` | Next option |
| `Shift+Tab` / `Left` / `Up` | Previous option |
| `Enter` | Select focused option |
| Hotkey (e.g. `s`, `d`) | Select option directly |
| `Esc` | Cancel |

## Known Limitations

<Aside type="caution">
- Modal width is determined by content — there is no explicit width control.
- Requires screen size to be set via `SetSize()` for correct centering.
- Mouse click support requires `tea.WithMouseCellMotion()` on the `tea.Program`.
</Aside>

### Error Variables

| Variable | Description |
|----------|-------------|
| `ErrModal` | Sentinel error for the `teamodal` package |
| `ErrCancelled` | Returned when the modal is cancelled |
| `ErrInvalidBounds` | Returned on invalid position bounds |
