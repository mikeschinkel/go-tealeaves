> **Module:** `github.com/mikeschinkel/go-tealeaves/teamodal` · **Type:** `ConfirmModel`

## Install

```bash
go get github.com/mikeschinkel/go-tealeaves/teamodal
```

Binary Yes/No and single-button OK dialogs for confirmations and alerts. Renders as a centered overlay on top of your application content using ANSI-aware compositing. Supports keyboard and mouse interaction, custom button labels, and styled borders.

## Quick Example

{/* verified: site/examples/confirm-dialog/compile_test.go:12 */}
```go
// Create a Yes/No confirmation
modal := teamodal.NewYesNoModal("Save changes before exiting?", &teamodal.ConfirmModelArgs{
    Title:      "Unsaved Changes",
    DefaultYes: true,
    YesLabel:   "Save",
    NoLabel:    "Discard",
})

// Open it
m.modal, cmd = m.modal.Open()

// In View() — overlay on background
if m.modal.IsOpen() {
    return m.modal.OverlayModal(backgroundView)
}

// Handle responses
case teamodal.AnsweredYesMsg:
    // User confirmed
case teamodal.AnsweredNoMsg:
    // User cancelled
```

### OK Modal

Single-button acknowledgment dialog:

{/* verified: site/examples/confirm-dialog/compile_test.go:25 */}
```go
modal := teamodal.NewOKModal("Operation completed successfully.", &teamodal.ConfirmModelArgs{
    Title:   "Done",
    OKLabel: "Got it",
})
```

Sends `ClosedMsg` when the user presses Enter or Esc.

## Features

- ANSI-aware overlay compositing via `OverlayModal()`
- Button focus with Tab/Shift+Tab and arrow keys
- Mouse click support for buttons (requires `tea.WithMouseCellMotion()`)
- Custom styling for title, message, buttons, border, and focus states
- Non-nil `tea.Cmd` message consumption signal (see [Modal Message Consumption](/go-tealeaves/guides/modal-consumption/))
- Multiline message support with `\n`
- Auto-centering on screen based on `SetSize()` dimensions

### ModalKeyMap Fields

`ConfirmModel` exposes a `Keys ModalKeyMap` public field. The key map controls dialog navigation:

| Field | Default keys | Description |
|-------|-------------|-------------|
| `Confirm` | `enter` | Activate the focused button |
| `Cancel` | `esc` | Cancel (same as No) |
| `NextButton` | `tab`, `right` | Move to the next button |
| `PrevButton` | `shift+tab`, `left` | Move to the previous button |
| `SelectLeft` | `left` | Move left to previous button |
| `SelectRight` | `right` | Move right to next button |

## Configuration

{/* verified: site/examples/confirm-dialog/compile_test.go:34 */}
```go
modal := teamodal.NewYesNoModal("message", &teamodal.ConfirmModelArgs{
    Title:              "Title",
    DefaultYes:         true,
    YesLabel:           "Confirm",
    NoLabel:            "Cancel",
    // Alignment (optional — defaults to lipgloss.Center)
    TextAlign:          lipgloss.Left,   // sets both TitleAlign and MessageAlign
    ButtonAlign:        lipgloss.Center,
    // Style overrides (optional)
    TitleStyle:         lipgloss.NewStyle().Bold(true),
    MessageStyle:       lipgloss.NewStyle(),
    ButtonStyle:        lipgloss.NewStyle(),
    FocusedButtonStyle: lipgloss.NewStyle().Bold(true),
    BorderStyle:        lipgloss.NewStyle(),
})

// Apply a theme (sets all style fields at once)
modal = modal.WithTheme(myTheme)

// Wither methods for post-construction updates
modal = modal.
    WithTitle("New Title").
    WithMessage("New message").
    WithTextAlign(lipgloss.Left).
    SetSize(width, height)
```

Uses the [Wither Methods](/go-tealeaves/guides/wither-methods/) pattern for configuration.
`WithTheme()` accepts a `teautils.Theme` and applies `theme.Modal.*` style fields. Individual `With*Style()` calls made after `WithTheme()` take precedence.

| `Enter` | Activate focused button |
| `Esc` | Cancel (same as No) |

## Known Limitations

<Aside type="caution">
- Modal width is determined by content — there is no explicit width control.
- Requires screen size to be set via `SetSize()` for correct centering. Call it on `tea.WindowSizeMsg`.
- Mouse click support requires `tea.WithMouseCellMotion()` on the `tea.Program`.
- The `OverlayModal()` function assumes the background view fills the full screen.
</Aside>
