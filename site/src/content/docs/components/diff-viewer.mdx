> **Module:** `github.com/mikeschinkel/go-tealeaves/teadiff` · **Types:** `SplitDiffModel`, `TUIRenderer` · **Install:** `go get github.com/mikeschinkel/go-tealeaves/teadiff`

Rendering diffs in a terminal requires synchronized scrolling, accurate ANSI-aware width measurement, and sensible keyboard navigation. `teadiff` provides two modes: an interactive `SplitDiffModel` for side-by-side comparison, and a `TUIRenderer` for condensed inline display.

<Aside type="note" title="Replaces teadiffview">
`teadiff` supersedes `teadiffview`. All condensed diff types (`TUIRenderer`, `DiffRenderer`, `FileDiff`, `CondensedBlock`, `FileStatus`) are available here unchanged, plus the new `SplitDiffModel`. See [migrating from teadiffr](/go-tealeaves/migration/teadiffr/).
</Aside>

<Aside type="caution" title="Provisional API">
This package is provisional as of v0.1.0. The public API may change in minor releases until promoted to stable.
</Aside>

## SplitDiffModel — interactive side-by-side diff

`SplitDiffModel` displays two diff panes side-by-side with synchronized horizontal and vertical scrolling, a cursor row, line selection, and optional gutter annotations.

### Quick example

{/* verified: site/examples/diff-viewer/ — go test passes */}
```go
model := teadiff.NewSplitDiffModel(&teadiff.SplitDiffModelArgs{
    Width:  termWidth,
    Height: termHeight,
    // Optional: syntax highlighting
    HighlightFunc: func(text, lang string) string {
        return highlighter.Highlight(text, lang)
    },
})

// Provide diff content (git-agnostic)
model = model.SetContent(diffContent) // diffutils.DiffContent

// In Update() — the model handles all keyboard events internally
model, cmd = model.Update(msg)

// In View()
return model.View()
```

### Keyboard shortcuts

| Key | Action |
|-----|--------|
| `↑` / `k` | Move cursor up |

| `Home` | Scroll to top |
| `End` | Scroll to bottom |

| `Ctrl+A` | Cycle: select block → select all → deselect all |

### Row annotations

Add visual indicators in the gutter column alongside any row:

{/* verified: site/examples/diff-viewer/ — go test passes */}
```go
annotations := map[int]teadiff.RowAnnotation{
    42: {Char: '●', Color: teacolor.Coral},  // Line 42: red dot
    17: {Char: '!', Color: teacolor.Gold},   // Line 17: warning
}
model = model.SetAnnotations(annotations)
```

### Pane line types

Each `SplitPaneRow` (returned by `Rows()`) holds two `PaneLine` values — one per side. Three concrete types implement `PaneLine`:

```
type SplitPaneRow struct {
    CommitLine PaneLine // Left pane (old/commit version)
    ActualLine PaneLine // Right pane (new/actual version)
    BlockIndex int      // 0 = unchanged; >0 = change block (1-based)
    LineOffset int      // Offset within the block
}
```

| Type | Interface | Description |
|------|-----------|-------------|
| `PaneLine` | `LineNo() int`, `PaneLine()` | Interface implemented by all pane cell types |
| `TextLine` | implements `PaneLine` | Real text content — `Text string` field, `LineNo()` for display line number |
| `BlockMarker` | implements `PaneLine` | Marks the start of a change block on the opposite side — `LineCount int` field |
| `PlaceholderLine` | implements `PaneLine` | Blank alignment row inserted to keep both sides in sync — `HunkLine int` field |

#### Constructors

| Function | Signature | Description |
|----------|-----------|-------------|
| `NewTextLine` | `NewTextLine(lineNo int, text string) *TextLine` | Create a text cell |
| `NewBlockMarker` | `NewBlockMarker(lineNo int, lineCount int) *BlockMarker` | Create a block-start marker |
| `NewPlaceholderLine` | `NewPlaceholderLine(lineNo int, hunkLine int) *PlaceholderLine` | Create an alignment placeholder |
| `(*BlockMarker).IsBlockStart` | `IsBlockStart() bool` | Reports whether this marker represents the start of a change block on the opposite pane |

### SelectAction

`SelectAction` tracks the Ctrl+A toggle cycle used by `ToggleBlockSelection()`:

| Constant | Value | Description |
|----------|-------|-------------|
| `SelectActionNone` | 0 | No selection active |
| `SelectActionBlock` | 1 | Current change block selected |
| `SelectActionAllBlocks` | 2 | All change blocks selected |

## TUIRenderer — condensed diff display

`TUIRenderer` renders a list of `FileDiff` values in a compact format suitable for non-interactive display — for example in a status pane or log view.

Two constructors are available:

| Function | Signature | Description |
|----------|-----------|-------------|
| `NewTUIRenderer` | `NewTUIRenderer(args *TUIRendererArgs) *TUIRenderer` | Create with explicit colors; pass `nil` for built-in defaults |
| `NewThemedTUIRenderer` | `NewThemedTUIRenderer(theme teautils.Theme) *TUIRenderer` | Create with colors derived from a `teautils.Theme` palette (maps `TextMuted`, `AccentAlt`, `StatusSuccess`, `StatusError`, `TintPositive`, `TintNegative`) |

### CommitGroupColors and SingleCommitColor

`CommitGroupColors` is a pre-defined `[]color.Color` of 8 distinct colors (green, blue, orange, purple, cyan, red, yellow, brown) intended for use as gutter annotation colors when visualizing per-commit-group line assignments. Use with `SetAnnotations()`.

`SingleCommitColor` is a single green (`#4CAF50`) color for use when all annotated rows belong to the same commit group.

{/* verified: site/examples/diff-viewer/ — go test passes */}
```go
annotations := map[int]teadiff.RowAnnotation{}
for lineIdx, groupIdx := range lineGroups {
    annotations[lineIdx] = teadiff.RowAnnotation{
        Char:  '●',
        Color: teadiff.CommitGroupColors[groupIdx%len(teadiff.CommitGroupColors)],
    }
}
model = model.SetAnnotations(annotations)
```

`NewTUIRenderer` accepts an optional `*TUIRendererArgs` to override specific colors. Pass `nil` to use the built-in defaults:

{/* verified: site/examples/diff-viewer/ — go test passes */}
```go
type TUIRendererArgs struct {
    FileHeaderColor    color.Color
    BlockHeaderColor   color.Color
    ContextColor       color.Color
    AddedColor         color.Color
    DeletedColor       color.Color
    NewStatusColor     color.Color
    DeletedStatusColor color.Color
    NewBgColor         color.Color
    DeletedBgColor     color.Color
}
```

### Quick example

{/* verified: site/examples/diff-viewer/ — go test passes */}
```go
// Pass nil for default colors, or &teadiff.TUIRendererArgs{...} to override specific colors
renderer := teadiff.NewTUIRenderer(nil)

diffs := []teadiff.FileDiff{
    {
        Path:   "cmd/main.go",
        Status: teadiff.FileModified,
        Blocks: []teadiff.CondensedBlock{
            {Type: "added", LineCount: 3, ChangedLines: []string{"+func foo() {}"}},
        },
    },
}

lines := teadiff.RenderFileDiffs(diffs, renderer, termWidth)
output := strings.Join(lines, "\n")
```

### FileDiff and CondensedBlock types

{/* verified: site/examples/diff-viewer/ — go test passes */}
```go
type FileDiff struct {
    Path   string
    Status FileStatus    // FileModified, FileNew, FileDeleted
    Blocks []CondensedBlock
}

type CondensedBlock struct {
    Type          string   // "added" or "deleted"
    LineCount     int
    ContextBefore []string
    ChangedLines  []string
    ContextAfter  []string
    IsTruncated   bool
}
```

### RenderFileDiffs

`RenderFileDiffs` is a standalone function that drives a `DiffRenderer` over a slice of `FileDiff` values and returns rendered lines:

{/* verified: site/examples/diff-viewer/ — go test passes */}
```go
func RenderFileDiffs(files []FileDiff, renderer DiffRenderer, width int) []string
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `files` | `[]FileDiff` | Diff data to render |
| `renderer` | `DiffRenderer` | Rendering strategy (e.g. `*TUIRenderer`) |
| `width` | `int` | Column width for line formatting |

Returns a `[]string` of styled lines ready to join with newlines or pass to a scroll pane.

{/* verified: site/examples/diff-viewer/ — go test passes */}
```go
renderer := teadiff.NewTUIRenderer(nil)
lines := teadiff.RenderFileDiffs(diffs, renderer, termWidth)
output := strings.Join(lines, "\n")
```

### DiffRenderer interface

Implement `DiffRenderer` to customize rendering for any output medium:

{/* verified: site/examples/diff-viewer/ — go test passes */}
```go
type DiffRenderer interface {
    RenderFileHeader(path string, status FileStatus, width int) string
    RenderBlockHeader(blockType string, lineCount int) string
    RenderContextLine(line string, status FileStatus, width int) string
    RenderAddedLine(line string, status FileStatus, width int) string
    RenderDeletedLine(line string, status FileStatus, width int) string
    RenderTruncation(status FileStatus) string
    RenderSeparator() string
}
```
