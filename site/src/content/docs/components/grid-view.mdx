> **Module:** `teagrid` · **Type:** `GridModel` · **Install:** `go get github.com/mikeschinkel/go-tealeaves/teagrid`

Displaying tabular data in the terminal with sorting, filtering, pagination, and row selection requires extensive custom code. `teagrid` provides a ready-made data grid component that handles all of these concerns out of the box, letting you focus on your data rather than table rendering logic.

## Install

```bash
go get github.com/mikeschinkel/go-tealeaves/teagrid
```

## Quick example

```go
package main

import (
	"os"
	"strings"

	tea "github.com/charmbracelet/bubbletea"
	"github.com/charmbracelet/lipgloss"
	"github.com/mikeschinkel/go-tealeaves/teagrid"
)

const (
	columnKeyName    = "name"
	columnKeyElement = "element"
)

func main() {
	m := model{
		table: teagrid.NewGridModel([]teagrid.Column{
			teagrid.NewColumn(columnKeyName, "Name", 13),
			teagrid.NewColumn(columnKeyElement, "Element", 10),
		}).WithBaseStyle(lipgloss.NewStyle().Align(lipgloss.Left)).WithRows([]teagrid.Row{
			teagrid.NewRow(teagrid.RowData{columnKeyName: "Pikachu", columnKeyElement: "Electric"}),
			teagrid.NewRow(teagrid.RowData{columnKeyName: "Charmander", columnKeyElement: "Fire"}),
		}),
	}
	p := tea.NewProgram(m)
	if _, err := p.Run(); err != nil {
		os.Exit(1)
	}
}

type model struct{ table teagrid.GridModel }

func (m model) Init() tea.Cmd { return nil }
func (m model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
	var cmd tea.Cmd
	m.table, cmd = m.table.Update(msg)
	if msg, ok := msg.(tea.KeyMsg); ok && msg.String() == "q" {
		return m, tea.Quit
	}
	return m, cmd
}
func (m model) View() string {
	var b strings.Builder
	b.WriteString("Simple data grid\nPress q to quit\n\n")
	b.WriteString(m.table.View())
	return b.String()
}
```

## Features

- Column definitions with key, title, width, and flex factor
- Fixed and flexible column widths
- Multi-column sorting (ascending/descending with secondary sort)
- Built-in fuzzy filtering with custom `FilterFunc` support
- Pagination with page size control
- Row selection (single and multi-select)
- Row style functions for conditional formatting
- Header and footer visibility toggles
- Keyboard highlight navigation

## API Reference

### Constructors

- **`NewGridModel(columns []Column) GridModel`** — Create a grid with the given column definitions.
- **`NewColumn(key, title string, width int) Column`** — Create a column with a key, display title, and fixed width.
- **`NewFlexColumn(key, title string, flexFactor int) Column`** — Create a flexible-width column that fills available space. Multiple flex columns share space proportional to their flex factors.
- **`NewRow(data RowData) Row`** — Create a row from a map of column keys to values.
- **`NewStyledCell(data any, style lipgloss.Style) StyledCell`** — Create a `StyledCell` (alias for `CellValue`) with a static style.
- **`NewStyledCellWithStyleFunc(data any, fn StyledCellFunc) StyledCell`** — Create a styled cell with a dynamic style function.

### Key types

| Type | Description |
|------|-------------|
| `GridModel` | Main grid component implementing `tea.Model` |
| `Column` | Column definition with key, title, width, and style |
| `Row` | Row data with `RowData` map and optional `Style` |
| `RowData` | `map[string]any` mapping column keys to values |
| `CellValue` | Rich cell value with optional style, dynamic style func, sort key, and spans |
| `Span` | A styled text segment within a cell (for rich inline text) |
| `CellStyleInput` | Context passed to `CellStyleFunc` (row, column, cursor state, metadata) |
| `CellStyleFunc` | `func(CellStyleInput) lipgloss.Style` — dynamic per-cell style |
| `Border` | Border character definitions |
| `SortColumn` | Sort specification (`ColumnKey`, `Direction`) |
| `FilterFuncInput` | Input for custom filter functions |
| `UserEvent` | Grid events (filter, highlight, row select changes) |

## Configuration

Configure the grid using `With*()` wither methods. These return the updated model rather than mutating in place (see [Wither Methods](/go-tealeaves/guides/wither-methods/)).

- **`WithRows(rows []Row)`** — Set row data.
- **`WithColumns(cols []Column)`** — Update column definitions.
- **`WithPageSize(n int)`** — Set rows per page.
- **`WithTargetWidth(w int)`** — Set target total width for the grid.
- **`WithBaseStyle(s lipgloss.Style)`** — Set the base cell style applied to all cells.
- **`WithFilterFunc(fn FilterFunc)`** — Provide a custom filter function.
- **`WithFuzzyFilter()`** — Enable the built-in fuzzy filter.
- **`WithHeaderVisibility(v bool)`** — Show or hide the header row.
- **`WithFooterVisibility(v bool)`** — Show or hide the footer row.

## Sorting

Sort rows by one or more columns. Primary sort sets the main sort column; secondary sort breaks ties.

- **`SortByAsc(key string)`** / **`SortByDesc(key string)`** — Set the primary sort column and direction.
- **`ThenSortByAsc(key string)`** / **`ThenSortByDesc(key string)`** — Add a secondary sort column.
- **`ColumnSorting(key string)`** — Query the current sort state for a column.

## Pagination

Navigate through pages when the row count exceeds the page size.

- **`PageDown()`** / **`PageUp()`** — Move forward or backward one page.
- **`PageFirst()`** / **`PageLast()`** — Jump to the first or last page.
- **`CurrentPage()`** / **`MaxPages()`** — Query the current page index and total page count.

## Row selection

Enable row selection to let users pick one or more rows from the grid.

- **`RowSelectToggle()`** — Toggle selection on the currently highlighted row.
- **`SelectedRows()`** — Get the list of currently selected rows.
- **`SelectableRows(b bool)`** — Enable or disable row selection.

## Rich cell values

Use `CellValue` instead of a plain string when you need per-cell styling, rich inline text, or a separate sort key.

### CellValue constructors

| Function | Description |
|----------|-------------|
| `NewCellValue(data, style)` | Cell with a static lipgloss style |
| `NewCellValueWithStyleFunc(data, fn)` | Cell with a `CellStyleFunc` for dynamic styling |
| `NewCellValueWithSortKey(data, sortValue, style)` | Cell with separate display value and sort key |
| `NewCellValueWithSpans(spans, style)` | Cell with rich inline text (multiple styled segments) |
| `NewSpan(text, style)` | Create a single `Span` within a `NewCellValueWithSpans` call |

### CellStyleFunc — dynamic per-cell styling

```go
// Highlight cells in the "status" column red when the value is "ERROR"
status := teagrid.NewCellValueWithStyleFunc("ERROR", func(in teagrid.CellStyleInput) lipgloss.Style {
    if in.IsHighlightedRow {
        return lipgloss.NewStyle().Bold(true).Foreground(lipgloss.Color("196"))
    }
    return lipgloss.NewStyle().Foreground(lipgloss.Color("196"))
})
```

`CellStyleInput` fields:

| Field | Type | Description |
|-------|------|-------------|
| `Data` | `any` | Cell's data value |
| `Column` | `Column` | Column definition |
| `Row` | `Row` | Row containing this cell |
| `RowIndex` | `int` | Row's position in the visible rows |
| `ColumnIndex` | `int` | Column's position |
| `IsHighlightedRow` | `bool` | True when the row cursor is on this row |
| `IsColCursor` | `bool` | True when the column cursor is on this cell |
| `GlobalMetadata` | `map[string]any` | Grid-level metadata set via `WithMetadata()` |

### Rich inline text (Spans)

```go
// A cell with a green check prefix and gray text
cell := teagrid.NewCellValueWithSpans([]teagrid.Span{
    teagrid.NewSpan("✓ ", lipgloss.NewStyle().Foreground(lipgloss.Color("82"))),
    teagrid.NewSpan("Complete", lipgloss.NewStyle().Foreground(lipgloss.Color("246"))),
}, lipgloss.Style{})
```

### SortValue — separate display and sort key

```go
// Display "Jan 1" in the cell, but sort by the underlying Unix timestamp
cell := teagrid.NewCellValueWithSortKey(
    "Jan 1",       // display value
    time.Unix(...), // sort value (comparable by <)
    lipgloss.Style{},
)
```

## Horizontal scrolling

For wide tables that exceed the terminal width, `teagrid` supports horizontal scrolling with optional frozen columns.

```go
// Freeze the first column so it stays visible while scrolling
grid = grid.WithHorizontalFreezeColumnCount(1)

// Scroll in key handler
case tea.KeyMsg:
    switch msg.String() {
    case "left":
        m.grid = m.grid.ScrollLeft()
    case "right":
        m.grid = m.grid.ScrollRight()
    }
```

| Method | Description |
|--------|-------------|
| `ScrollLeft()` | Scroll viewport one column to the left |
| `ScrollRight()` | Scroll viewport one column to the right |
| `WithHorizontalFreezeColumnCount(n)` | Freeze the first n columns (they remain visible during horizontal scroll) |

## Per-row dynamic styling

Use `WithRowStyleFunc` for styling that depends on row data (e.g., alternating row colors, status-based highlights):

```go
grid = grid.WithRowStyleFunc(func(in teagrid.RowStyleFuncInput) lipgloss.Style {
    if in.IsHighlighted {
        return lipgloss.NewStyle().Background(lipgloss.Color("235"))
    }
    return lipgloss.Style{}
})
```

## Grid-level metadata

Attach context to the grid that is passed to every `CellStyleFunc` call via `CellStyleInput.GlobalMetadata`:

```go
grid = grid.WithMetadata(map[string]any{
    "selectedIDs": selectedSet,
})
```

<Aside type="caution" title="Known limitations">
- Column widths must be specified at creation time. There is no auto-width calculation from cell content.
</Aside>
