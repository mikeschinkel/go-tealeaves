> **Module:** `teafields` · **Type:** `DropdownModel` · **Install:** `go get github.com/mikeschinkel/go-tealeaves/teafields`

Bubble Tea has no built-in dropdown component. Building one from scratch requires overlay compositing that respects ANSI escape codes, intelligent above/below positioning based on available terminal space, keyboard and mouse input handling, and scrollable option lists. `teafields` handles all of this in a single, composable component that follows the Tea Leaves [Modal Message Consumption](/go-tealeaves/guides/modal-consumption/) pattern.

## Install

```bash
go get github.com/mikeschinkel/go-tealeaves/teafields
```

## Quick example

{/* verified: site/examples/dropdown-control/ — go test passes */}
```go
package main

import (
	"fmt"
	"os"

	tea "github.com/charmbracelet/bubbletea"
	"github.com/mikeschinkel/go-tealeaves/teafields"
)

func main() {
	items := teafields.ToOptions([]string{"Apple", "Banana", "Cherry", "Date", "Elderberry"})
	dropdown := teafields.NewDropdownModel(items, &teafields.DropdownModelArgs{
		FieldRow: 3,
		FieldCol: 18,
	})

	p := tea.NewProgram(model{dropdown: dropdown}, tea.WithAltScreen())
	if _, err := p.Run(); err != nil {
		fmt.Fprintf(os.Stderr, "Error: %v\n", err)
		os.Exit(1)
	}
}
```

## Features

- Intelligent above/below positioning based on available terminal space
- ANSI-aware overlay compositing via `OverlayDropdown()`
- Keyboard navigation (up/down/enter/esc)
- Scrollable option list when items exceed visible area
- Non-nil `tea.Cmd` consumption signal (see [Modal Message Consumption](/go-tealeaves/guides/modal-consumption/) pattern)
- Configurable border and item styles
- Terminal size auto-detection

## API Reference

### Constructors and functions

| Function | Signature | Description |
|----------|-----------|-------------|
| `NewDropdownModel` | `NewDropdownModel(options []Option, args *DropdownModelArgs) DropdownModel` | Create a dropdown with the given options and configuration |
| `ToOptions` | `ToOptions(items []string) []Option` | Convert a string slice into a slice of `Option` values |
| `OverlayDropdown` | `OverlayDropdown(background, foreground string, row, col int) string` | ANSI-aware overlay compositing of foreground onto background |
| `EnsureTermGetSize` | `EnsureTermGetSize(fd uintptr)` | Initialize terminal size detection for above/below positioning |

### Key types

| Type | Description |
|------|-------------|
| `DropdownModel` | Main dropdown component — implements `tea.Model` |
| `DropdownKeyMap` | Key bindings configuration (Cancel, Down, Select, Up) |
| `Option` | A selectable option with `Text` and `Value` fields |
| `OptionSelectedMsg` | Message sent when the user selects an option — contains `Index`, `Text`, and `Value` fields |
| `DropdownModelArgs` | Configuration arguments for `NewDropdownModel` |
| `ModelArgs` | Type alias for `DropdownModelArgs` (deprecated — use `DropdownModelArgs`) |

### Default Style Functions

| Function | Signature | Description |
|----------|-----------|-------------|
| `DefaultBorderStyle` | `DefaultBorderStyle() lipgloss.Style` | Default border style for the dropdown container |
| `DefaultItemStyle` | `DefaultItemStyle() lipgloss.Style` | Default style for unselected items |
| `DefaultSelectedStyle` | `DefaultSelectedStyle() lipgloss.Style` | Default style for the highlighted item |

### DropdownKeyMap fields

`DropdownKeyMap` configures which keys control the dropdown. Override it by assigning to the `Keys` field if your application uses different bindings.

| Field | Default keys | Description |
|-------|-------------|-------------|
| `Up` | `up`, `k` | Move selection up one item |
| `Down` | `down`, `j` | Move selection down one item |
| `Select` | `enter` | Confirm and select the highlighted item |
| `Cancel` | `esc` | Dismiss the dropdown without selecting |

{/* verified: site/examples/dropdown-control/ — go test passes */}
```go
km := teafields.DefaultDropdownKeyMap()
km.Up = key.NewBinding(key.WithKeys("ctrl+p"), key.WithHelp("ctrl+p", "move up"))
dropdown.Keys = km
```

### DropdownModelArgs fields

| Field | Type | Description |
|-------|------|-------------|
| `FieldRow` | `int` | Row position of the triggering field (dropdown appears just below/above it) |
| `FieldCol` | `int` | Column position of the triggering field |
| `ScreenWidth` | `int` | Terminal width (used for above/below and overflow detection) |
| `ScreenHeight` | `int` | Terminal height (used for above/below placement decision) |
| `TopMargin` | `int` | Minimum row to position above (e.g. `1` to avoid a menu bar) |
| `BottomMargin` | `int` | Rows reserved at bottom (e.g. `1` to avoid a status bar) |
| `BorderStyle` | `lipgloss.Style` | Border style for the dropdown container (optional) |
| `ItemStyle` | `lipgloss.Style` | Style for unselected items (optional) |
| `SelectedStyle` | `lipgloss.Style` | Style for the highlighted item (optional) |

### DropdownModel Wither Methods

| Method | Signature | Description |
|--------|-----------|-------------|
| `WithOptions` | `WithOptions(items []Option) DropdownModel` | Replace the option list |
| `WithTopMargin` | `WithTopMargin(margin int) DropdownModel` | Set the top margin (rows reserved above) |
| `WithBottomMargin` | `WithBottomMargin(margin int) DropdownModel` | Set the bottom margin (rows reserved below) |
| `WithTheme` | `WithTheme(theme teautils.Theme) DropdownModel` | Apply `DropdownTheme` styles from a `teautils.Theme` |
| `Keys` | `Keys DropdownKeyMap` | Public field — assign directly to override key bindings |

### DefaultDropdownKeyMap

| Function | Signature | Description |
|----------|-----------|-------------|
| `DefaultDropdownKeyMap` | `DefaultDropdownKeyMap() DropdownKeyMap` | Return the default key bindings for the dropdown |

### Error Variables

| Variable | Description |
|----------|-------------|
| `ErrDropdown` | Sentinel error for the `teafields` package |
| `ErrEmptyOptions` | Returned when the options slice is empty |
| `ErrInvalidBounds` | Returned on invalid position bounds |
| `ErrInvalidCol` | Returned on invalid column value |
| `ErrInvalidIndex` | Returned on invalid option index |
| `ErrInvalidRow` | Returned on invalid row value |

## Configuration

`DropdownModel` uses the [Wither Methods](/go-tealeaves/guides/wither-methods/) pattern for configuration:

{/* verified: site/examples/dropdown-control/ — go test passes */}
```go
m.dropdown = m.dropdown.
    WithScreenSize(msg.Width, msg.Height).
    WithPosition(row, col)
```

| Method | Description |
|--------|-------------|
| `WithPosition(row, col int)` | Set the dropdown's row and column position |
| `WithScreenSize(width, height int)` | Set screen dimensions for above/below auto-detection |
| `Open()` | Open the dropdown (returns model and `tea.Cmd`) |
| `Close()` | Close the dropdown (returns model and `tea.Cmd`) |

## Messages

| Message | When sent |
|---------|-----------|
| `OptionSelectedMsg` | User selects an option — contains `Index`, `Text`, and `Value` fields |
| `DropdownCancelledMsg` | User presses `esc` to cancel without selecting — struct with no fields |

{/* verified: site/examples/dropdown-control/ — go test passes */}
```go
case teafields.OptionSelectedMsg:
    // User picked an option
    fmt.Println(msg.Text, msg.Value)
case teafields.DropdownCancelledMsg:
    // User cancelled — close any parent overlay
```

## Keyboard shortcuts

| Key | Action |
|-----|--------|
| `up` / `k` | Move selection up |
| `down` / `j` | Move selection down |
| `enter` | Select current option |
| `esc` | Cancel and close |

## Known limitations

<Aside type="caution">
- Dropdown width is determined by the widest option text — there is no explicit width control.
- Mouse click support requires `tea.WithMouseCellMotion()` on the `tea.Program`.
</Aside>
