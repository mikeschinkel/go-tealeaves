> **Module:** `teadiffr` · **Types:** `TUIRenderer`, `DiffRenderer` · **Install:** `go get github.com/mikeschinkel/go-tealeaves/teadiffr`

`teadiffr` renders lists of file diffs in a compact, styled format for non-interactive terminal display — useful in status panes, log views, or anywhere you need to show condensed diff output alongside other UI.

<Aside type="caution" title="Deprecated — migrate to teadiff">
`teadiffr` is superseded by [`teadiff`](/go-tealeaves/components/diff-viewer/). All types (`TUIRenderer`, `DiffRenderer`, `FileDiff`, `CondensedBlock`, `FileStatus`) and `RenderFileDiffs` are available in `teadiff` with identical names and signatures, plus the new `SplitDiffModel` for interactive side-by-side viewing. See the [migration guide](/go-tealeaves/migration/teadiffr/).
</Aside>

## Install

```bash
go get github.com/mikeschinkel/go-tealeaves/teadiffr
```

## Quick example

{/* verified: site/examples/diff-renderer/ — go test passes */}
```go
import "github.com/mikeschinkel/go-tealeaves/teadiffr"

// Build diff data
diffs := []teadiffr.FileDiff{
    {
        Path:   "cmd/main.go",
        Status: teadiffr.FileModified,
        Blocks: []teadiffr.CondensedBlock{
            {
                Type:         "added",
                LineCount:    2,
                ChangedLines: []string{"func newHandler() http.Handler {", "    return mux"},
            },
        },
    },
    {
        Path:   "internal/config.go",
        Status: teadiffr.FileNew,
        Blocks: []teadiffr.CondensedBlock{
            {Type: "added", LineCount: 10, ChangedLines: []string{"package internal"}},
        },
    },
}

// Create renderer with defaults
renderer := teadiffr.NewTUIRenderer(nil)

// Render to a slice of styled strings
lines := teadiffr.RenderFileDiffs(diffs, renderer, termWidth)

// Join and display
output := strings.Join(lines, "\n")
fmt.Println(output)
```

## Key types

### FileDiff and CondensedBlock

{/* verified: site/examples/diff-renderer/ — go test passes */}
```go
type FileDiff struct {
    Path   string
    Status FileStatus       // FileModified, FileNew, FileDeleted
    Blocks []CondensedBlock
}

type CondensedBlock struct {
    Type          string   // "added" or "deleted"
    LineCount     int
    ContextBefore []string
    ChangedLines  []string
    ContextAfter  []string
    IsTruncated   bool
}
```

### FileStatus

| Constant | Meaning |
|----------|---------|
| `FileModified` | File was changed |
| `FileNew` | File was created |
| `FileDeleted` | File was removed |

## TUIRenderer

`TUIRenderer` is the built-in lipgloss-backed renderer. Construct it with `NewTUIRenderer`:

{/* verified: site/examples/diff-renderer/ — go test passes */}
```go
// Default colors
renderer := teadiffr.NewTUIRenderer(nil)

// Override specific colors
renderer := teadiffr.NewTUIRenderer(&teadiffr.TUIRendererArgs{
    AddedColor:   lipgloss.Color("46"),   // bright green
    DeletedColor: lipgloss.Color("196"),  // bright red
})
```

### TUIRendererArgs fields

| Field | Default (ANSI 256) | Description |
|-------|--------------------|-------------|
| `FileHeaderColor` | `244` (gray) | File path label |
| `BlockHeaderColor` | `135` (violet) | Block type + line count |
| `ContextColor` | `240` (dark gray) | Unchanged context lines |
| `AddedColor` | `34` (green) | Added line foreground |
| `DeletedColor` | `160` (red) | Deleted line foreground |
| `NewStatusColor` | `46` (bright green) | `[NEW]` status tag |
| `DeletedStatusColor` | `168` (pink) | `[DELETED]` status tag |
| `NewBgColor` | `22` (dark green) | Background for new-file blocks |
| `DeletedBgColor` | `52` (dark red) | Background for deleted-file blocks |

### Themed renderer

If your app uses the Tea Leaves theming system, use `NewThemedTUIRenderer` to derive colors from your palette automatically:

{/* verified: site/examples/diff-renderer/ — go test passes */}
```go
import "github.com/mikeschinkel/go-tealeaves/teautils"

theme := teautils.DefaultTheme()
renderer := teadiffr.NewThemedTUIRenderer(theme)
```

The themed constructor maps palette slots (`StatusSuccess`, `StatusError`, `TintPositive`, `TintNegative`, etc.) to the corresponding renderer color fields.

## DiffRenderer interface

Implement `DiffRenderer` to customize rendering for any output medium — HTML, plain text, a different styling library, etc.:

{/* verified: site/examples/diff-renderer/ — go test passes */}
```go
type DiffRenderer interface {
    RenderFileHeader(path string, status FileStatus, width int) string
    RenderBlockHeader(blockType string, lineCount int) string
    RenderContextLine(line string, status FileStatus, width int) string
    RenderAddedLine(line string, status FileStatus, width int) string
    RenderDeletedLine(line string, status FileStatus, width int) string
    RenderTruncation(status FileStatus) string
    RenderSeparator() string
}
```

Pass any `DiffRenderer` implementation to `RenderFileDiffs`:

{/* verified: site/examples/diff-renderer/ — go test passes */}
```go
lines := teadiffr.RenderFileDiffs(diffs, myCustomRenderer, width)
```

## RenderFileDiffs

`RenderFileDiffs` is the top-level rendering function. It iterates files and blocks, delegating to the renderer for each element:

{/* verified: site/examples/diff-renderer/ — go test passes */}
```go
func RenderFileDiffs(files []FileDiff, renderer DiffRenderer, width int) []string
```

Files are separated by the renderer's `RenderSeparator()` output followed by a blank line. Within each file, blocks appear in order with context lines surrounding changed lines. Truncated blocks emit `RenderTruncation` at the end.

## API Reference

### Functions

| Function | Signature | Description |
|----------|-----------|-------------|
| `RenderFileDiffs` | `RenderFileDiffs(files []FileDiff, renderer DiffRenderer, width int) []string` | Render a slice of file diffs using the given renderer; returns styled lines |
| `NewTUIRenderer` | `NewTUIRenderer(args *TUIRendererArgs) *TUIRenderer` | Create a renderer with optional color overrides; pass `nil` for defaults |
| `NewThemedTUIRenderer` | `NewThemedTUIRenderer(theme teautils.Theme) *TUIRenderer` | Create a renderer with colors derived from a `teautils.Theme` |

### DiffRenderer Interface

| Method | Signature | Description |
|--------|-----------|-------------|
| `RenderFileHeader` | `RenderFileHeader(path string, status FileStatus, width int) string` | Render the file header line |
| `RenderBlockHeader` | `RenderBlockHeader(blockType string, lineCount int) string` | Render the block header (type + count) |
| `RenderContextLine` | `RenderContextLine(line string, status FileStatus, width int) string` | Render an unchanged context line |
| `RenderAddedLine` | `RenderAddedLine(line string, status FileStatus, width int) string` | Render an added line |
| `RenderDeletedLine` | `RenderDeletedLine(line string, status FileStatus, width int) string` | Render a deleted line |
| `RenderTruncation` | `RenderTruncation(status FileStatus) string` | Render truncation indicator |
| `RenderSeparator` | `RenderSeparator() string` | Render separator between files |

### FileDiff Struct

| Field | Type | Description |
|-------|------|-------------|
| `Path` | `string` | File path |
| `Status` | `FileStatus` | `FileModified`, `FileNew`, or `FileDeleted` |
| `Blocks` | `[]CondensedBlock` | Condensed change blocks |

### FileStatus Constants

| Const | Description |
|-------|-------------|
| `FileModified` | File was changed |
| `FileNew` | File was created |
| `FileDeleted` | File was removed |

### CondensedBlock Struct

| Field | Type | Description |
|-------|------|-------------|
| `Type` | `string` | `"added"` or `"deleted"` |
| `LineCount` | `int` | Total number of changed lines |
| `ContextBefore` | `[]string` | Unchanged lines before the change |
| `ChangedLines` | `[]string` | The added or deleted lines |
| `ContextAfter` | `[]string` | Unchanged lines after the change |
| `IsTruncated` | `bool` | Whether the block was truncated |
