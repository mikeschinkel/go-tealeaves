Common utilities like key registries, centering calculations, and border helpers get reimplemented in every Bubble Tea project. `teautils` provides a shared set of utilities organized into three areas: a centralized key binding registry, positioning and measurement functions, and display rendering helpers.

## Install

```bash
go get github.com/mikeschinkel/go-tealeaves/teautils
```

## Key Registry

The key registry provides centralized management of key bindings with namespace validation, category grouping, and integration with help modals and status bars.

<Aside type="tip">
For a complete walkthrough including registration patterns, namespace validation, and integration with teastatus, see the [Key Registry pattern](/go-tealeaves/patterns/key-registry/) page.
</Aside>

```go
registry := teautils.NewKeyRegistry()

registry.MustRegister(teautils.KeyMeta{
    ID:             teautils.MustParseKeyIdentifier("sys.help"),
    Binding:        key.NewBinding(key.WithKeys("?"), key.WithHelp("?", "Help")),
    StatusBar:      true,
    StatusBarLabel: "Help",
    HelpModal:      true,
    Category:       "System",
    HelpText:       "Toggle help overlay",
})
```

### Registry methods

| Method | Description |
|--------|-------------|
| `NewKeyRegistry() *KeyRegistry` | Create a new empty registry |
| `Register(meta KeyMeta) error` | Add or update a key binding with metadata |
| `MustRegister(meta KeyMeta)` | Like `Register` but panics on error |
| `RegisterMany(metas []KeyMeta) error` | Add multiple key bindings (fails on first error) |
| `MustRegisterMany(metas []KeyMeta)` | Like `RegisterMany` but panics on error |
| `Get(id KeyIdentifier) (*KeyMeta, error)` | Retrieve a key by its identifier |
| `Clear()` | Remove all keys (useful when switching views) |
| `ForStatusBar() []KeyMeta` | Get keys marked for status bar display, in registration order |
| `ForHelpModal() []KeyMeta` | Get keys marked for help modal display |
| `ByCategory() map[string][]KeyMeta` | Get keys grouped by category (keys without category go under "Other") |

## Positioning utilities

Functions for centering modals and measuring rendered views.

```go
// Measure a rendered view (ANSI-aware)
width, height := teautils.MeasureRenderedView(renderedModal)

// Calculate centered position
row, col := teautils.CalculateCenter(screenW, screenH, width, height)

// Or do both at once (with a slight upward shift for visual balance)
width, height, row, col := teautils.CenterModal(renderedModal, screenW, screenH)
```

### Positioning functions

| Function | Signature | Description |
|----------|-----------|-------------|
| `CalculateCenter` | `CalculateCenter(screenW, screenH, modalW, modalH int) (row, col int)` | Compute row/col to center a modal on screen (clamped to 0) |
| `MeasureRenderedView` | `MeasureRenderedView(renderedView string) (width, height int)` | Measure ANSI-aware width and line count of a rendered string |
| `CenterModal` | `CenterModal(renderedView string, screenW, screenH int) (width, height, row, col int)` | Measure and center in one call (shifts up 1 row for visual balance) |

## Display helpers

Functions for rendering styled, aligned content and formatting key labels.

### Rendering

| Function | Signature | Description |
|----------|-----------|-------------|
| `RenderAlignedLine` | `RenderAlignedLine(text string, style lipgloss.Style, width int, align lipgloss.Position) string` | Render styled text with horizontal alignment within a width |
| `RenderCenteredLine` | `RenderCenteredLine(text string, style lipgloss.Style, width int) string` | Convenience wrapper for center-aligned rendering |
| `ApplyBoxBorder` | `ApplyBoxBorder(borderStyle lipgloss.Style, content string) string` | Apply a border with `Padding(1, 2)` to content |

### Key display formatting

| Function | Signature | Description |
|----------|-----------|-------------|
| `FormatKeyDisplay` | `FormatKeyDisplay(k KeyMeta) string` | Format key bindings as `[Key1]/[Key2]` display strings |
| `ProperCaseShortcut` | `ProperCaseShortcut(s string) string` | Title-case each part of a shortcut: `ctrl+c` becomes `Ctrl+C` |
| `GetSortedCategories` | `GetSortedCategories(keysByCategory map[string][]KeyMeta, preferredOrder []string) []string` | Sort categories by preferred order, then alphabetically |

### Key identifier functions

| Function | Signature | Description |
|----------|-----------|-------------|
| `ParseKeyIdentifier` | `ParseKeyIdentifier(s string) (KeyIdentifier, error)` | Validate and create a `KeyIdentifier` from a string |
| `MustParseKeyIdentifier` | `MustParseKeyIdentifier(s string) KeyIdentifier` | Like `ParseKeyIdentifier` but panics on error |

## Key types

| Type | Description |
|------|-------------|
| `KeyRegistry` | Centralized registry for key bindings and display metadata |
| `KeyMeta` | Key binding metadata: `ID`, `Binding`, `Category`, `HelpText`, `StatusBar`, `StatusBarLabel`, `HelpModal`, `DisplayKeys` |
| `KeyIdentifier` | Dot-separated namespace string (e.g., `"nav.up"`, `"action.delete"`) |
| `HelpVisorStyle` | Styling for help visor rendering: `TitleStyle`, `CategoryStyle`, `KeyStyle`, `DescStyle`, `KeyColumnGap`, `CategoryOrder` |

## Help visor styling

The `HelpVisorStyle` type configures how help overlays are rendered. It is used by components that display categorized key help (such as `teamodal.ListModel`).

```go
style := teautils.DefaultHelpVisorStyle()

// Customize
style.TitleStyle = lipgloss.NewStyle().Bold(true).Foreground(lipgloss.Color("99"))
style.CategoryOrder = []string{"Navigation", "Actions", "System", "Other"}
style.KeyColumnGap = 4
```

## Known limitations

<Aside type="caution">
- `KeyIdentifier` validation requires dot-separated namespace format (e.g., `"nav.up"`). Single-segment identifiers like `"quit"` are rejected -- use `"sys.quit"` instead.
- `MeasureRenderedView` counts ANSI-aware character widths but does not account for terminal-specific font rendering differences.
</Aside>
