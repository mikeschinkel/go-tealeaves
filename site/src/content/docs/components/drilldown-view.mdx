> **Module:** `github.com/mikeschinkel/go-tealeaves/teatree` · **Type:** `DrillDownModel[T]` · **Install:** `go get github.com/mikeschinkel/go-tealeaves/teatree`

Dependency graphs are complex — navigating them interactively requires tree traversal with circular dependency detection, path display from root to leaf, and alternative selection at each level. `DrillDownModel[T]` (in the `teatree` module) provides an interactive path viewer that renders a single root-to-leaf path through a node tree, with dropdown selection for choosing sibling alternatives at any level. It integrates `teafields` internally for the dropdown and uses a configurable `DrillDownSelectorFunc` to determine the "best" path.

## Install

```bash
go get github.com/mikeschinkel/go-tealeaves/teatree
```

## Quick example

{/* verified: site/examples/drilldown-view/ — go test passes */}
```go
import "github.com/mikeschinkel/go-tealeaves/teatree"

// Build a node tree
root := teatree.NewNode("myapp", "My App", myAppData)
libA := teatree.NewNode("libA", "Library A", libAData)
libB := teatree.NewNode("libB", "Library B", libBData)
libC := teatree.NewNode("libC", "Library C", libCData)

libA.SetChildren([]*teatree.Node[MyData]{libC})
root.SetChildren([]*teatree.Node[MyData]{libA, libB})

// Create the drilldown model
viewer := teatree.NewDrillDownModel(root, teatree.DrillDownArgs[MyData]{
    SelectorFunc: func(parent *teatree.Node[MyData], children []*teatree.Node[MyData]) (*teatree.Node[MyData], error) {
        return children[0], nil // always pick first child
    },
    Prompt: "Dependency path:",
})

// Initialize (builds initial path from root to leaf)
viewer, err := viewer.Initialize()
if err != nil {
    log.Fatal(err)
}

// In Update():
viewer, cmd = viewer.Update(msg)

// Handle outbound messages
case teatree.DrillDownSelectMsg[MyData]:
    // User confirmed a leaf node with Enter
    doSomethingWith(msg.Node)
case teatree.DrillDownChangeMsg[MyData]:
    // User picked a different sibling from the dropdown

// In View():
return viewer.View()
```

## Features

- Path-based navigation through node trees (root to leaf)
- Dropdown selection for sibling alternatives at each level (via `teafields` internally)
- Custom `DrillDownSelectorFunc[T]` for ranking which child to follow at each level
- Configurable border, path, and selected item styles
- Optional prompt text above the path display
- Optional blank separator line between prompt and first item

### DrillDownSelectorFunc[T]

`DrillDownSelectorFunc[T]` is `func(current *Node[T], children []*Node[T]) (*Node[T], error)`. It is called at each level to select which child node to follow when building the initial root-to-leaf path. Returns an error to abort initialization.

## The SelectorFunc

The `DrillDownSelectorFunc[T]` determines which child to follow when building the initial path from root to leaf:

{/* verified: site/examples/drilldown-view/ — go test passes */}
```go
selector := func(parent *teatree.Node[MyData], children []*teatree.Node[MyData]) (*teatree.Node[MyData], error) {
    // Pick the child with the deepest subtree
    best := children[0]
    bestDepth := countDepth(best)
    for _, child := range children[1:] {
        if d := countDepth(child); d > bestDepth {
            best = child
            bestDepth = d
        }
    }
    return best, nil
}
```

The function can capture any context it needs via closure.

<Aside type="caution">
`Initialize()` must be called after `NewDrillDownModel()` before the model can be used. The initial path is built lazily to allow configuration before display.
</Aside>
