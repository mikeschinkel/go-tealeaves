> **Module:** `github.com/mikeschinkel/go-tealeaves/teanotify` · **Type:** `NotifyModel` · **Install:** `go get github.com/mikeschinkel/go-tealeaves/teanotify`

Toast/notification UIs in the terminal require overlay rendering, timed auto-dismiss, and multiple positioning options. Building this from scratch for each project is wasteful. `teanotify` provides a ready-made notification component that handles all of these concerns, letting you focus on your application logic.

## Install

```bash
go get github.com/mikeschinkel/go-tealeaves/teanotify
```

## Quick example

{/* verified: site/examples/notification-view/compile_test.go:12 */}
```go
// Create notification model
notify := teanotify.NewNotifyModel(teanotify.NotifyOpts{
    Width:           40,
    Duration:        3 * time.Second,
    AllowEscToClose: true,
})
notify, err := notify.Initialize()

// Send a notification
cmd := m.notify.NewNotifyCmd(teanotify.InfoKey, "File saved successfully")

// In View() — render overlay on content
return m.notify.Render(fullContent)
```

## Features

- 4 built-in notice types: Info, Warn, Error, Debug
- Custom notice types via `RegisterNoticeType()`
- 6 screen positions (TopLeft, TopCenter, TopRight, BottomLeft, BottomCenter, BottomRight)
- Auto-dismiss with configurable duration
- Color fade animation using linear interpolation
- ESC key to dismiss early (configurable)
- Unicode prefix symbols (configurable)
- ANSI-aware overlay rendering via `Render()`
- Minimum width configuration

### Two-Step Initialization

`NotifyModel` requires a two-step initialization pattern:

1. `NewNotifyModel(opts)` — creates the model with configuration
2. `Initialize() (NotifyModel, error)` — registers built-in notice types and validates configuration; **must be called before use**

This is different from the single-step `Init() tea.Cmd` that satisfies `tea.Model`. Call `Initialize()` once at startup, then use `Init()` as part of the normal Bubble Tea lifecycle.

{/* verified: site/examples/notification-view/compile_test.go:12 */}
```go
notify := teanotify.NewNotifyModel(opts)
notify, err := notify.Initialize() // required before use
```

## Configuration

Configure the notification using `With*()` wither methods. These return the updated model rather than mutating in place (see [Wither Methods](/go-tealeaves/guides/wither-methods/)).

- **`WithPosition(pos Position)`** — Change position at runtime.
- **`WithMinWidth(w int)`** — Set minimum notification width.
- **`WithUnicodePrefix(b bool)`** — Toggle Unicode prefix symbols.
- **`WithAllowEscToClose(b bool)`** — Toggle ESC dismissal.


## Custom notice types

Register additional notice types beyond the four built-ins using `RegisterNoticeType()`. It returns the updated model and an error if validation fails (e.g. nil color):

{/* verified: site/examples/notification-view/compile_test.go:40 */}
```go
notify, err := notify.RegisterNoticeType(teanotify.NoticeDefinition{
    Key:       "success",
    Prefix:    "[✓]",
    ForeColor: "#00ff87", // bright green (ANSI 46)
})
if err != nil {
    log.Fatal(err)
}

// Use the custom key like any built-in
cmd := notify.NewNotifyCmd("success", "Build complete")
```

`RegisterNoticeType()` has immutable semantics — it returns a new model with the extra type registered, leaving the receiver unchanged.

## Keyboard shortcuts

| Key | Action |
|-----|--------|
| `esc` | Dismiss active notification (when `AllowEscToClose` is true) |

<Aside type="caution" title="Known limitations">
- Only one notification displayed at a time — new notifications replace the current one.
- Color fade requires terminal with true color support for best results.
</Aside>
