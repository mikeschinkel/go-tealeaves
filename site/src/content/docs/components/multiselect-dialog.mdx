## Install

```bash
go get github.com/mikeschinkel/go-tealeaves/teamodal
```

Generic multi-select checkbox list modal. Items must implement the `MultiSelectItem` interface. All items start checked by default; users toggle individual items before pressing a button. Supports scrollable lists, optional footer text, and hotkeys on buttons.

## Quick Example

{/* verified: site/examples/multiselect-dialog/ ÔÇö go test passes */}
```go
// Define an item type that implements MultiSelectItem
type FileItem struct {
    id   string
    name string
}
func (f FileItem) ID() string    { return f.id }
func (f FileItem) Label() string { return f.name }

// Create and open the modal
modal := teamodal.NewMultiSelectModel([]FileItem{
    {id: "main.go",   name: "main.go"},
    {id: "util.go",   name: "util.go"},
    {id: "config.go", name: "config.go"},
}, &teamodal.MultiSelectModelArgs{
    Title:   "Select files to delete",
    Message: "The following files will be permanently removed.",
    Footer:  "This action cannot be undone.",
    Buttons: []teamodal.MultiSelectButton{
        {Label: "Delete", Hotkey: 'd', ID: "delete"},
    },
})

// Open it
m.modal, cmd = m.modal.Open()

// In View()
if m.modal.IsOpen() {
    return m.modal.OverlayModal(backgroundView)
}

// Handle responses
case teamodal.MultiSelectButtonPressedMsg[FileItem]:
    switch msg.ButtonID {
    case "delete":
        for _, f := range msg.Selected {
            // f is a FileItem ÔÇö only the checked items
        }
    }
case teamodal.MultiSelectCancelledMsg:
    // User pressed Esc
```

## Features

- Generic over any type implementing `MultiSelectItem` ÔÇö no type assertions needed
- Checkbox list with `[Ôťô]` / `[ ]` indicators; all items start checked by default
- Scrollable viewport with configurable `MaxVisible` (default 8) and scrollbar
- Two-area focus: list navigation and button row, cycled with Tab / Shift+Tab
- Button hotkeys (case-insensitive) trigger button press from any focus state
- Auto-appended Cancel button unless `NoCancel: true`
- Optional `Title`, `Message`, and `Footer` text areas
- ANSI-aware overlay compositing via `OverlayModal()`
- Non-nil `tea.Cmd` message consumption signal (see [Modal Message Consumption](/go-tealeaves/guides/modal-consumption/))
- Auto-centering on screen based on `SetSize()` dimensions

| Type | Description |

| `Logger` | `*slog.Logger` | Optional logger for debugging; set via `WithLogger()` |

| Field | Default keys | Description |

### MultiSelectModelArgs Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `Title` | `string` | `""` | Title text shown at the top of the modal |
| `Message` | `string` | `""` | Optional description shown above the list |
| `Footer` | `string` | `""` | Optional note shown below the list (e.g. warning) |
| `Buttons` | `[]MultiSelectButton` | `nil` | Action buttons (2ÔÇô4 recommended) |
| `AllChecked` | `bool` | `true` | Whether all items start checked |
| `MaxVisible` | `int` | `8` | Maximum items visible; list scrolls beyond this |
| `NoCancel` | `bool` | `false` | Suppress the auto-appended Cancel button |
| `CancelEmitsButtonPressed` | `bool` | `false` | Make Cancel button emit `ButtonPressedMsg` instead of `CancelledMsg` |
| `ScreenWidth` | `int` | `0` | Initial screen width (updated by `WindowSizeMsg`) |
| `ScreenHeight` | `int` | `0` | Initial screen height (updated by `WindowSizeMsg`) |

### Messages

| Message | When sent |
|---------|-----------|
| `MultiSelectButtonPressedMsg[T]` | A button was pressed ÔÇö contains `ButtonID` (string) and `Selected` ([]T, the checked items) |

## Known Limitations

<Aside type="caution">
- Modal width is determined by content ÔÇö there is no explicit width control.
- Requires screen size to be set via `SetSize()` or a `WindowSizeMsg` for correct centering.
- `SetItems` resets all checked state to fully-checked regardless of the original `AllChecked` value.
</Aside>

### Error Variables

| Variable | Description |

| `ErrCancelled` | Returned when the modal is cancelled |
| `ErrInvalidBounds` | Returned on invalid position bounds |
